## Prerequisites

- Azure account (free tier works)
- Microsoft Teams (personal or work account)
- Node.js 22+
- Claude Code CLI installed

## 1. Create Azure Bot Registration

1. Go to [Azure Portal](https://portal.azure.com) → Create a resource → "Azure Bot"
2. Fill in:
   - **Bot handle**: choose a unique name
   - **Pricing tier**: F0 (free) for development
   - **Microsoft App ID**: Create new
   - **App type**: Single Tenant
3. Click Create

### Switch to Multi Tenant

1. Go to [App Registrations](https://portal.azure.com/#view/Microsoft_AAD_RegisteredApps/ApplicationsListBlade) → find your app
2. Manifest → set `signInAudience` to `AzureADandPersonalMicrosoftAccount`
3. Save

> Required for personal Microsoft accounts. Without this, only users in the same Azure AD tenant can use the bot.

### Get credentials

1. Go to [App Registrations](https://portal.azure.com/#view/Microsoft_AAD_RegisteredApps/ApplicationsListBlade) → find your app
2. Overview → copy the **Application (client) ID** and **Directory (tenant) ID**
3. Certificates & secrets → New client secret → copy the **Value** (this is your App Password)

## 2. Create Dev Tunnel

```bash
devtunnel create --id my-teams-bot --allow-anonymous
devtunnel port create my-teams-bot -p 3978
```

Set the messaging endpoint in [Azure Bot Configuration](https://portal.azure.com/#view/HubsExtension/BrowseResource/resourceType/Microsoft.BotService%2FbotServices):

Your Bot → Settings → Configuration → Messaging endpoint:
```
https://my-teams-bot-3978.devtunnels.ms/api/messages
```

## 3. Install & Configure

See [README](../README.md#quick-start) for installation, then:

```bash
teams-bot setup
```

Enter the credentials from step 1 when prompted. Setup will automatically generate `teams-claude-bot.zip` for the next step.

## 4. Register in Teams

### Option A: Upload manifest zip

Upload the `teams-claude-bot.zip` (generated by setup) to Teams Admin Center → Manage apps → Upload

To regenerate: `teams-bot package`

### Option B: Teams Developer Portal

1. Go to [Teams Developer Portal](https://dev.teams.microsoft.com/apps)
2. Import app → select `manifest/manifest.json`
3. Update the Bot ID field with your App ID
4. Preview in Teams

## 5. Install & Start

```bash
teams-bot install    # Build + install as background service
```

Send any message to the bot in Teams to activate — this stores your conversation ID for handoff.

## 6. Verify

- Send "hello" to the bot in Teams — you should get a response from Claude Code
- Run `teams-bot status` to check the service is running
- Run `teams-bot logs` to see output

## Troubleshooting

### Bot doesn't respond
- Check `teams-bot status` — is the service running?
- Check `teams-bot logs` — any errors?
- Verify messaging endpoint URL in Azure Portal matches your tunnel URL
- Ensure tunnel is active: `devtunnel host my-teams-bot`

### "Unauthorized" errors
- Verify App ID, Password, and Tenant ID are correct
- Re-run `teams-bot setup` to update credentials

### "Upload failed due to an invalid BotId in your manifest"
- The `botId` in manifest must be the **Application (client) ID** (a UUID) from [App Registrations](https://portal.azure.com/#view/Microsoft_AAD_RegisteredApps/ApplicationsListBlade) → your app → Overview
- Re-run `teams-bot setup` with the correct ID, then `teams-bot package` to regenerate the zip
- Or fix it directly in [Teams Developer Portal](https://dev.teams.microsoft.com/apps) → your app → App features → Bot → paste the correct ID

### Tunnel auth expired
```bash
devtunnel user logout
devtunnel user login
teams-bot restart
```
